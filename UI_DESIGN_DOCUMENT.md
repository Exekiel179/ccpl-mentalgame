## 1. 设计原则

### 1.1 核心设计语言

| 原则 | 说明 |
|------|------|
| **心理安全感** | 暖色系底色 + 柔和圆角，避免强对比和刺激性视觉元素 |
| **层次清晰** | 三层信息层级：主体内容 > 辅助信息 > 背景装饰 |
| **呼吸感** | 充足的留白（Margin 24~48px），密度适中，不产生压迫感 |
| **可读性优先** | 正文最小字号 14px，行间距不低于 1.4 倍，关键信息使用 Semi-Bold |
| **一致性** | 所有面板使用统一圆角半径（24px）和阴影规格 |

### 1.2 情感基调

> 暖奶油 + 莫兰迪绿 + 黏土橙 = 温暖、治愈、专业而不冰冷

| 场景 | 颜色值 | 说明 |
|------|--------|------|
| 游戏失败背景 | `Color(0.03, 0.05, 0.13)` | 深夜蓝，营造沉重感 |

- **禁止**直接使用 `Color.RED` / `Color.BLUE` 等饱和原色作为界面颜色
- 所有透明度调整使用 `.a` 属性，不创建新的颜色值
- 暗色背景（游戏结算页）例外，使用专属配色

## 3. 字体系统

### 3.1 字体栈（优先级降序）

```
Source Han Sans SC → Noto Sans CJK SC → Microsoft YaHei
→ Yu Gothic → Segoe UI → Sans-Serif
```

### 3.2 渲染设置

```gdscript
font.antialiasing          = TextServer.FONT_ANTIALIASING_LCD
font.hinting               = TextServer.HINTING_LIGHT
font.subpixel_positioning  = TextServer.SUBPIXEL_POSITIONING_ONE_HALF
font.generate_mipmaps      = true
```

### 3.3 字重规范

| 字重 | `font_weight` | 使用场景 |
|------|--------------|---------|
| Regular | 400 | 正文说明、列表内容 |
| Semi-Bold / Bold | 700 | 标题、关键词、按钮文字、数字 |

### 3.4 字号规范

| 级别 | 字号 | 字重 | 典型用途 |
|------|------|------|---------|
| XL 标题 | 52px | Bold | 词语高亮（语音校准当前词） |
| L 标题 | 32~36px | Bold | 页面主标题 |
| M 标题 | 22~28px | Bold | 二级标题、卡片标题 |
| S 标题 | 18~20px | Bold | 三级标题、按钮文字 |
| 正文 | 16~17px | Regular | 说明文字、卡片内容 |
| 辅助 | 14px | Regular/Bold | 徽章、标签、元数据 |
| 微型 | 12px | Regular | 卡片编号、版权信息 |

## 4. 组件规范

### 4.1 面板（Panel）

**标准卡片面板**：

```
圆角半径：24px（所有角）
背景色：COLOR_PANEL_BG（白色 92% 透明度）
边框：1px，COLOR_PANEL_BORDER
阴影：12px 扩散，Y 偏移 4px，Color(0,0,0, 0.05)
内边距：24~32px（根据内容密度调整）
```

**轻量面板**（提示卡等）：

```
圆角半径：16px
背景色：COLOR_PANEL_BG
边框：1px，COLOR_PANEL_BORDER
内边距：12~16px
```

### 4.2 按钮（Button）

#### 主操作按钮（Filled）

```
最小尺寸：140 × 44px
圆角半径：24px
背景色：功能色（如 COLOR_MORANDI_GREEN）
文字颜色：Color.WHITE
字重：Bold 16px
内边距左右：20px
悬停态：背景色 lightened(0.1)
按下态：背景色 darkened(0.1)
```

#### 次要按钮（Ghost / Outline）

```
背景色：Color.TRANSPARENT
边框：2px，颜色与文字同
文字颜色：同边框色
悬停态：背景轻微填充 Color(1,1,1, 0.1)
按下态：背景加深 Color(0,0,0, 0.1)
```

#### 特殊按钮规格

| 按钮类型 | 尺寸 | 背景色 | 字号 |
|---------|------|--------|------|
| 主功能按钮（FeatureButton） | 自适应 × 54px | COLOR_CLAY_ORANGE | 22px |
| 难度选择按钮 | 170 × 56px | 灰白 → 选中变 MORANDI_GREEN | 14px |
| 操作按钮（标准） | 140 × 44px | 功能色 | 16px |
| 小按钮（试听等） | 56 × 28px | 功能色 | 12px |

#### 按钮交互反馈

所有按钮需注册悬停反馈：
```gdscript
# 鼠标进入：缩放到 1.05
create_tween().tween_property(btn, "scale", Vector2(1.05, 1.05), 0.2)
    .set_trans(Tween.TRANS_SINE)
# 鼠标离开：恢复 1.0
create_tween().tween_property(btn, "scale", Vector2(1.0, 1.0), 0.2)
    .set_trans(Tween.TRANS_SINE)
```

`pivot_offset` 需在 `resized` 信号触发时同步更新为 `size * 0.5`。

### 4.3 进度条（ProgressBar）

```
高度：8px
圆角：4px（背景和填充均设置）
背景色：COLOR_WARM_BG
填充色：根据用途选择功能色
  - 健康值：COLOR_MORANDI_GREEN
  - 计时器：COLOR_CLAY_ORANGE
  - 压力值：COLOR_MIST_BLUE
禁用默认百分比文字显示（show_percentage = false）
```

### 4.4 波形显示（SampleWaveform）

```
最小高度：36px
分段数：24 根竖线
线宽：3px
颜色：COLOR_MORANDI_GREEN（已采集）/ Color(0.8, 0.8, 0.8)（占位）
振幅：归一化到容器高度 40%（上下各 20%）
```

### 4.5 矢量图标（VectorIcon）

内置四种手绘风格图标，纯 `draw_*` 方法实现：

| 类型 | 代码键 | 用途 |
|------|--------|------|
| 麦克风 | `Type.MIC` | 录音状态指示 |
| 播放 | `Type.PLAY` | 试听按钮 |
| 对勾 | `Type.CHECK` | 验证通过指示 |
| 波浪 | `Type.WAVE` | 采集进行中 / 样本粒子动画 |

默认尺寸：24 × 24px，颜色可配置。

### 4.6 圆形录音球（Recording Orb）

```
形状：圆形（corner_radius 所有角设为 1000）
尺寸：最小高度 120px，长宽比 1:1
边框：4px，Color.WHITE
背景：COLOR_PANEL_BG
阴影：40px 扩散，COLOR_MORANDI_GREEN（透明度 0.2）
录音中：脉冲动画 1.0→1.1→1.0，0.8秒/循环（Tween.TRANS_SINE）
停止录音：立即杀死 Tween，恢复 scale = Vector2.ONE
```

## 6. 动效规范

### 6.1 全局动效原则

| 原则 | 说明 |
|------|------|
| 有意义 | 动效须服务于信息传达，不做无目的装饰 |
| 克制 | 持续时间控制在 0.15~0.6 秒内 |
| 缓动 | 使用 `TRANS_SINE` 或 `TRANS_CUBIC`，避免线性动画 |
| 响应感 | 用户交互反馈（按钮按下、卡牌选中）应在 100ms 内响应 |

### 6.2 动效参数速查

| 效果 | 时长 | 缓动 | 属性 |
|------|------|------|------|
| 按钮悬停放大 | 0.20s | TRANS_SINE | scale 1.0→1.05 |
| 按钮离开恢复 | 0.20s | TRANS_SINE | scale 1.05→1.0 |
| 暂停菜单入场 | 0.18s（scale）+ 0.14s（alpha） | TRANS_BACK | scale 0.88→1.0，alpha 0→1 |
| 录音球脉冲（单次） | 0.40s | TRANS_SINE | scale 1.0→1.1（循环） |
| 样本粒子飞行 | 0.60s | TRANS_CUBIC, EASE_OUT | global_position→目标卡片 |
| 粒子消失 | 0.60s（同时） | TRANS_QUAD, EASE_IN | scale→0.2 |
| 场景切换淡入 | 0.30s | — | 由 Godot 默认处理 |

### 6.3 卡牌句子动画

```
出生动画：从随机边缘沿追踪曲线移向玩家
追踪物理：TURN_RATE = 1.4，弹簧跟随
摆动效果：spine_pos 直行，card_body 横向 ±10px 正弦摆动
超时消失：淡出 + 缩小（0.3s）
正确消失：绿色光晕 + 向上飞散（0.4s）
错误消失：红色抖动 + 爆炸颗粒（0.5s）
```

### 6.4 CBT 卡牌拖拽

```
拖拽跟随：目标位置弹簧物理（STIFFNESS = 500.0，DAMPING = 0.35）
释放归位：弹性回弹到原位（TRANS_ELASTIC，1.2s）
放入插槽：抖动确认动画（0.2s scale 1.0→1.1→1.0）
```

### 9.3 代码中统一使用方式

优先从主题文件加载样式，无法从主题加载时，在代码中动态创建 `StyleBoxFlat`，遵循上述面板规范。

```gdscript
# 标准面板样式（可复用 helper 方法）
func _make_panel_style(bg: Color, radius: int = 24) -> StyleBoxFlat:
    var sb := StyleBoxFlat.new()
    sb.bg_color = bg
    sb.set_corner_radius_all(radius)
    sb.border_width_left = 1
    sb.border_width_top = 1
    sb.border_width_right = 1
    sb.border_width_bottom = 1
    sb.border_color = COLOR_PANEL_BORDER
    sb.shadow_color = Color(0, 0, 0, 0.05)
    sb.shadow_size = 12
    sb.shadow_offset = Vector2(0, 4)
    return sb
```
