## 1. 项目概述

### 1.1 产品定位

**心理健康训练场** 是一款面向青少年与年轻成人的认知行为训练游戏（CBT-based Gamified Therapy）。核心目标是通过游戏化的交互机制，提升玩家的**认知重构（Cognitive Restructuring）**能力，降低日常生活中的心理困扰。

### 1.2 三大游戏模式

| 模式 | 名称 | 核心体验 | 训练目标 |
|------|------|---------|---------|
| 模式一 | 🌀 迷宫探索训练 | 快节奏动作分类 | 快速辨别"事实 vs 想法" |
| 模式二 | ☁ 知识连线配对 | 配对益智解谜 | 理解与记忆 CBT 核心概念 |
| 模式三 | 🃏 CBT 卡牌挑战 | 回合制叙事卡牌 | 系统练习七步认知重构流程 |

### 1.3 核心价值主张

| 维度 | 说明 |
|------|------|
| 教育层面（局外） | 系统介绍认知行为疗法知识，涵盖认知扭曲类型、重构技巧、情绪命名等 |
| 训练层面（局内） | 三种不同难度与节奏的游戏模式，覆盖从理解到应用的完整学习路径 |
| 场景化 | 覆盖学业压力、家庭矛盾、社交压力三类真实生活压力源 |
| 渐进性 | 三级难度 × 三个场景 × 三种模式，提供递进式训练路径 |

## 2. 核心设计理念

### 2.1 心理安全优先

游戏的视觉语言遵循**暖系治愈美学**：暖奶油底色、莫兰迪绿、黏土橙，避免刺激性高饱和配色，为玩家营造放松、安全的训练环境。

### 2.2 三种模式的训练层次

```
知识理解（连线配对）
    → 将概念名称与定义准确匹配
    → 低压力、可查阅、即时反馈

快速应用（迷宫探索）
    → 在时间压力下快速分类"事实 vs 想法"
    → 训练直觉反应，强化辨别速度

系统实践（CBT 卡牌）
    → 完整走完七步认知重构流程
    → 模拟真实生活场景的深度练习
```

### 2.3 渐进式暴露原则

三种游戏模式对应认知训练的不同阶段，玩家可自由选择，也可按阶梯顺序进行。难度系统在迷宫模式中进一步提供精细化的压力递进，与 CBT 暴露疗法（Exposure Therapy）原理对齐。

## 5. 玩法模式一：迷宫探索训练

### 5.1 概述

**类型**：俯视角动作分类游戏（Top-down Action + Judgment）
**关卡数**：3 关（难度递增）
**核心考验**：在时间压力和空间移动的同时，快速判断句子的认知属性
**灵感参考**：纸镜奇缘（Paper Mirror Journey）

### 5.2 关卡参数

| 关卡 | 基础时限 | 初始出牌间隔 | 最短出牌间隔 | 速率衰减 |
|------|----------|-------------|-------------|---------|
| 第一关 | 120 秒 | 4.0 秒 | 1.8 秒 | 0.04/秒 |
| 第二关 | 90 秒 | 3.0 秒 | 1.4 秒 | 0.05/秒 |
| 第三关 | 60 秒 | 2.0 秒 | 1.0 秒 | 0.06/秒 |

> 实际参数 = 基础参数 × 难度倍率

### 5.3 玩家系统

| 属性 | 数值 |
|------|------|
| 移动速度 | 150 px/s |
| 基础生命值 | 100 HP（随难度调整） |
| 操作方式 | WASD / 方向键移动 |

### 5.4 句子卡牌（Sentence Card）

#### 行为模型

```
生成点（四边随机）
    → 物理追踪玩家（弹簧物理，TURN_RATE = 1.4）
    → 幽灵摆动效果（卡脊直行，卡身左右摆动 ±10 px）
    → 倒计时 6 秒（超时自动消失，计为「躲避」）
    → 玩家接触 → 判定交互
```

#### 判定规则

| 操作 | 判定条件 | 结果 |
|------|---------|------|
| 鼠标左键 / "事实"语音 | 句子属于事实 | ✅ 正确 +10分（连击加成） |
| 鼠标右键 / "想法"语音 | 句子属于想法 | ✅ 正确 +10分（连击加成） |
| 分类错误 | 任意 | ❌ -20 HP，连击重置 |
| 未分类直接碰撞 | — | ❌ -15 HP（盾牌可抵消一次） |
| 超时未处理 | — | 卡牌消失，无惩罚 |

#### 移动参数

| 参数 | 数值 |
|------|------|
| 基础速度 | 80 ~ 140 px/s（随机范围） |
| 难度速度倍率 | 轻松 ×0.85 / 标准 ×1.0 / 进阶 ×1.15 |
| 追踪转向率 | 1.4 rad/s |

### 5.5 连击系统

| 连击数 | 奖励 |
|--------|------|
| 每次正确 | 积累连击，分数 = 10 + (连击-1)×5 |
| 3 连击 | 回复 5 HP |
| 5 连击 | 回复 10 HP |
| 7、9、11... 连击 | 回复 8 HP（奇数连击触发） |

### 5.6 道具系统

每关生成 5 个道具（固定比例分布）：

| 类型 | 数量 | 效果 |
|------|------|------|
| 治疗（HEAL） | 2 | 恢复 15 HP |
| 减速（SLOW） | 1 | 卡牌速度降为 50%，持续 8 秒 |
| 时间（TIME） | 1 | 增加 10 秒倒计时 |
| 护盾（SHIELD） | 1 | 抵消下一次碰撞伤害 |

### 5.7 迷宫与地图

- **生成方式**：程序化格子迷宫（`maze_builder.gd`）
- **探索机制**：玩家视野外有雾效遮罩（`fog_layer.gd`），需要探索地图
- **通关条件**：到达出口区域且剩余时间 > 0

### 5.8 HUD 布局

```
┌──────────────────────────────────────────┐
│ [心理健康] ████████░░ 75/100  │ 得分: 320  │
│ 进度条（左上）                 │ 📚学业压力  │
│                                │ ⏱ 01:23   │
└────────────────┬───────────────┴──────────┘
                 │
         [连击 x5!] ← 中心临时显示（连击≥2 时出现）
```

## 6. 玩法模式二：知识连线配对

### 6.1 概述

**类型**：配对益智游戏（Matching Quiz）
**关卡数**：每局 5 题，可刷新换一组
**核心考验**：将心理学概念的解释与正确的名称标题匹配连接
**定位**：低压力的知识理解与记忆训练，适合首次接触 CBT 的玩家

### 6.2 游戏机制

#### 基本流程

```
加载 5 张知识闪卡（从 ScenarioDatabase 随机抽取，不重复）
    ↓
显示当前题：中央展示某张卡的「详细解释」
    ↓
上方排列 5 个标题按钮（随机打乱顺序）
    ↓
玩家点击认为匹配的标题
    ↓
判定结果 → 进入下一题
    ↓
5 题全部完成 → 结算界面
```

#### 判定与反馈

| 操作结果 | 视觉反馈 | 文字提示 | 后续行为 |
|---------|---------|---------|---------|
| 选择正确 | 按钮变绿（success 态） | "答对啦，这张知识卡已经连好了。" | 0.4 秒后加载下一题 |
| 选择错误 | 按钮变红（error 态） | "这张还不太对，再看看下面的解释提示。" | 弹出详情 Modal，0.4 秒后按钮恢复 |

#### 辅助功能

- **右键标题按钮**：查看该卡的完整详情（不影响题目进度）
- **换一组按钮**：重新抽取 5 张不同的卡片，重置题目

### 6.3 数据说明

每道题使用的闪卡数据结构：

```
{
  "id":          唯一标识符
  "title":       概念名称（显示在标题按钮上）
  "detail":      详细解释（显示在中央配对区）
  "layer":       知识层级（如"心理学基础"）
  "scenario":    适用场景
  "tags":        标签数组
  "distortion":  对应的认知扭曲类型（若有）
  "example_text": 示例文本（可选）
}
```

### 6.4 完成后跳转逻辑

连线配对游戏完成后，根据进入时的来源模式决定跳转目标：

| 来源模式（`knowledge_return_mode`） | 完成按钮文字 | 跳转目标 |
|------------------------------------|------------|---------|
| `GameMode.CARD` | 开始卡牌训练 ▶ | CBT 卡牌挑战 |
| `GameMode.MAZE`（默认） | 开始迷宫训练 ▶ | 迷宫探索训练 |

> 连线配对也可作为独立模式从主菜单直接进入，无需衔接后续游戏。

### 6.5 防重复机制

`_previous_matching_ids` 记录本次会话中已出现过的卡片 ID，换题时排除已用卡片，保证每次刷新都能接触新内容。

### 6.6 UI 布局

```
┌─────────────────────────────────────┐
│  ☁ CBT 知识配对                     │
│  「请把下面的解释，连到上方正确的标题」│
│                                     │
│  [标题A] [标题B] [标题C] [标题D] [标题E]  ← 5个标题按钮
│  ─────────────────────────────────  │
│  第 X / 5 题                        │
│  ┌─────────────────────────────┐   │
│  │  当前题解释内容（可滚动）      │   │
│  └─────────────────────────────┘   │
│  反馈文字区域                        │
│                                     │
│  [换一组 ✨]  [开始游戏 ▶]  [返回菜单] │
└─────────────────────────────────────┘
```

## 7. 玩法模式三：CBT 卡牌挑战

### 7.1 概述

**类型**：回合制叙事卡牌游戏
**关卡数**：单局，跟随剧本走完完整治疗流程
**核心考验**：引导玩家完整走过 CBT 七步流程，系统练习认知重构
**灵感参考**：Slay the Spire 卡牌交互 + DBT/CBT 工作手册

### 7.2 七步治疗流程

| 步骤 | 阶段名称 | 玩家操作 | 说明 |
|------|---------|---------|------|
| 1 | **情境呈现** SITUATION | 阅读确认 | 展示危机情境 |
| 2 | **自动化想法** AUTOMATIC_THOUGHT | 选择卡牌 | 识别自动浮现的负性思维 |
| 3 | **情绪命名** EMOTION | 选择 + 拖拽强度条 | 标注情绪类型及强度（1-100） |
| 4 | **认知扭曲** DISTORTION | 选择扭曲类型 | 识别思维中的认知错误 |
| 5 | **寻找证据** EVIDENCE | 选择支持/反驳证据 | 基于事实检验想法 |
| 6 | **认知重构** REFRAME | 拖拽技能卡 | 应用技能卡构建更平衡的想法 |
| 7 | **重新评分** RERATE | 拖拽强度条 | 对比重构前后情绪强度变化 |

### 7.3 游戏属性

| 属性 | 初始值 | 最大值 | 说明 |
|------|--------|--------|------|
| 心理健康（HP） | 100 | 100 | 错误选择会扣减 |
| 压力值（Stress） | 32 | 100 | 重构成功后降低 |
| 洞察力（Insight） | 3 | — | 使用技能卡的货币，消耗 1 点/张 |

### 7.4 卡牌类型

| 类型 | 颜色 | 功能 |
|------|------|------|
| CRISIS（危机牌） | 棕褐色 | 代表扭曲的自动化想法 |
| DISTORTION（扭曲牌） | 橙色 | 标识认知错误类型 |
| EVIDENCE（证据牌） | 蓝色/绿色 | 提供支持或反驳证据 |
| SKILL（技能牌） | 绿色系 | 提供重构技术 |
| BALANCED_THOUGHT（平衡想法牌） | 绿色 | 健康的替代性想法 |

### 7.5 五大核心技能卡

| 技能名 | 针对扭曲类型 | 效果 |
|--------|------------|------|
| 认知重构 | 灾难化、非黑即白、标签化、过度概括 | 挑战极端化思维，寻找中间路径 |
| 寻找证据 | 预测未来、读心术、心理过滤 | 基于事实检验想法 |
| 自我关怀 | 应该陈述、标签化、个性化 | 以对待朋友的方式对待自己 |
| 换个角度 | 过度放大、非黑即白、忽视积极面 | 寻找不同的诠释视角 |
| 接纳情绪 | 情绪推理、过度放大 | 与情绪共存而非对抗 |

### 7.6 物理交互

- **卡牌拖拽**：弹簧物理（刚度 500.0，阻尼 0.35）
- **技能区域**：拖拽到指定区域后释放，触发重构动画
- **情绪强度**：左右拖拽滑块，动态更新数值

## 8. 语音识别系统

### 8.1 系统架构

```
麦克风输入（AudioStreamMicrophone）
    → VoiceCapture Bus（静音总线）
    → AudioEffectCapture（3秒缓冲区）
    → VoiceService 处理帧数据
    → VoiceClassifier（双方法分类器）
    → 输出：「事实」/ 「想法」/ 未识别
```

> 语音识别仅在迷宫模式中作为可选输入方式，连线和卡牌模式均为纯鼠标/键盘操作。

### 8.2 分类算法

**方法 1：全局统计特征（20 维向量，权重 55%）**
- 能量均值/标准差、过零率、高频内容
- 三频段能量（低/中/高）、峰值振幅
- 时域形状比（前半/后半能量比、ZCR 比）

**方法 2：DTW 时序对比（帧级 5 维特征，权重 45%）**
- 动态时间规整（Dynamic Time Warping），Sakoe-Chiba 带限
- 帧特征：能量、ZCR、高频、高频段能量、峰值

**置信度阈值**：< 25% 时返回空结果（未识别）

### 8.3 校准流程

```
阶段 1：COLLECTING（样本采集）
    → 录制「事实」× 4 次 + 「想法」× 4 次
    → 每次最长 1.8 秒，最短有效 0.3 秒

阶段 2：VERIFYING（稳定性验证）
    → 连续识别正确 2 次 → 通过该词，进入下一词
    → 识别有误 → 将该录音加入训练集 → 重新训练 → 继续验证
    → 累计失败 3 次 → 清空该词样本，从头采集该词

完成 → 保存模型（user://voice_templates_v3.dat）→ 进入游戏
```

### 8.4 操作映射（迷宫模式）

| 操作 | 触发方式 |
|------|---------|
| 开始录音 | SPACE 键或「开始录音」按钮 |
| 判定「事实」 | 语音识别 "事实" 或鼠标左键 |
| 判定「想法」 | 语音识别 "想法" 或鼠标右键 |

## 9. 知识卡片浏览系统

### 9.1 功能定位

知识卡片浏览（`knowledge_screen.gd`）是独立于三个游戏模式之外的**学习参考区**，供玩家在游戏前后浏览心理学知识。

### 9.2 展示形式

- 每次随机展示 **3 张**知识卡片（横向排列）
- 每张卡片支持点击翻转，查看标题和详细说明
- 支持刷新重新抽取
- 底部按钮可直接进入连线配对或迷宫/卡牌游戏

### 9.3 知识库规模

- 总计 **30+ 知识卡片**（`data/scenario_database.gd`）
- 覆盖：CBT 基础概念、12 种认知扭曲详解、场景化建议、危机支持资源

### 11.2 程序化音乐合成

`AmbientMusic` 通过 `AudioStreamGenerator` 实时生成谐波叠加音乐：
- **谐波倍音**：基频 + 2倍、3倍、4倍频叠加
- **四套预设**：各含不同频率组合，营造不同情绪氛围
- **淡入淡出**：场景切换时平滑过渡（0.5 秒）

### 13.3 场景切换协议

所有场景切换通过 `GameManager` 的方法调用，禁止在业务脚本中直接调用 `get_tree().change_scene_to_file()`：

```gdscript
GameManager.go_to_main_menu()
GameManager.start_game(scenario)          # 迷宫
GameManager.start_card_game(scenario)     # 卡牌
GameManager.go_to_knowledge(mode)         # 连线配对
GameManager.go_to_win()
GameManager.go_to_game_over()
```

### 13.4 GDScript 编码规范

- 使用 Godot 4 语法（无 Godot 3 兼容代码）
- 所有变量、参数、返回值必须类型标注
- `@export` 用于 Inspector 可见属性
- 信号连接使用 `.connect()` 方法（非字符串方式）
- 自定义类使用 `class_name` 声明（数据类）
- 场景内节点引用使用 `@onready var`
